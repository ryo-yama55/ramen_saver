## 📁 ファイル構成

```
works/
├── README.md       # このファイル（worksディレクトリの説明）
├── TODO.md         # 作業項目TODOリスト（詳細タスク定義）

```

---

## 📄 各ファイルの役割

### TODO.md
**用途**: 具体的な作業タスクの定義と管理

**内容**:
- WORK-001〜WORK-013 の詳細タスク定義
- 各タスクの優先度、見積時間、受け入れ基準
- チェックボックスによる進捗管理
- フェーズ別の分類（Phase 0〜3）

**使い方**:
1. 次に着手するタスクを確認する
2. タスク内のチェックボックスを更新しながら作業を進める
3. タスク完了時に全てのチェックボックスにチェックを入れる

**更新頻度**: 毎日

---

### PROGRESS.md
**用途**: プロジェクト全体の進捗状況を一目で確認

**内容**:
- Epic別の進捗率（視覚的なプログレスバー）
- 完了済み機能のリスト
- 進行中・未着手のタスク
- 既知の問題（バグ）
- マイルストーン管理
- 開発メモ

**使い方**:
1. プロジェクトの現在地を確認する
2. 週次・月次の振り返りに使用する
3. ステークホルダーへの報告に使用する

**更新頻度**: タスク完了時、週次

## 🔄 ワークフロー

### 1. 作業開始時
```bash
# 1. PROGRESS.mdで全体の進捗を確認
cat aidlc-docs/works/PROGRESS.md

# 2. TODO.mdで次のタスクを確認
cat aidlc-docs/works/TODO.md
```

### 2. 作業中
- TODO.md のチェックボックスを更新しながら進める
- 新しい課題が見つかったら TODO.md に追加

### 3. タスク完了時
1. TODO.md の該当タスクを全てチェック
2. PROGRESS.md の進捗率を更新
3. 完了機能リストに追加
4. GitHubのIssue/PRと連携

## 💡 Tips

### タスク見積の目安
- 🟢 小（1-2時間）: コンポーネント修正、小規模な機能追加
- 🟡 中（2-4時間）: 新規画面作成、複雑なロジック実装
- 🔴 大（4時間以上）: 複数画面の統合、アーキテクチャ変更

### チェックボックスの使い方
```markdown
- [ ] 未完了タスク
- [x] 完了タスク
```

### 進捗報告のテンプレート
```markdown
## 今週の成果
- WORK-XXX を完了（Epic X - US-X.X）
- WORK-YYY を50%完了

## 来週の予定
- WORK-YYY を完了予定
- WORK-ZZZ に着手予定

## ブロッカー
- なし / 〇〇が必要
```
